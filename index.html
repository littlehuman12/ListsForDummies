Lists

A list is a data structure (a way of organising data). A list has items of the same datatype and may include duplicates. The number of items in a list can grow or shrink. Items can be added into a particular position in a list.

One way to implement a list in Java to use an ArrayList. A number of different operations can be perfromed on an ArrayList. These are shown in the table below.

The examples are based on the following list with the identifier myList: 45, 66, 41, 13, 19, 8.

List Operation Description Example Output
isEmpty() Test for empty list myList.isEmpty() False
add(item) Appends the specified item to the end of a list. myList.add(10) 45, 66, 41, 13, 19, 8, 10
add(index, item) Inserts the specified item at the specified position index in a list. myList.add(1, 69) 45, 69, 66, 41, 13, 19, 8, 10
remove() Removes the element at the specified position in this list. Shifts any subsequent elements to the left (subtracts one from their indices). myList.remove(0) 69, 66, 41, 13, 19, 8, 10
size() Returns the number of elements in this list i.e the size of the list. myList.size() 69, 66, 41, 13, 19, 8, 10 so 7.
get(index) Gets the element of a specified position within the list. myList.get(5) 69, 66, 41, 13, 19, 8, 10 so 8.
clear() Removes all of the elements from this list. myList.clear() ()
contains(item) Returns true if this list contains the specified element. myList.contains(69) True
lastIndexOf(item) Returns the index in this list of the last occurrence of the specified element, or -1 if the list does not contain this element. myList.lastIndexOf(13) 69, 66, 41, 13, 19, 8, 10 so 3.
indexOf(item) Returns the index in this list of the first occurrence of the specified element, or -1 if the list does not contain this element. myList.indexOf(41) 69, 66, 41, 13, 19, 8, 10 so 2.
  • A collection of data items in a sequence without any gaps (unlike arrays).
  • Accessed via index (like arrays) starting at 0.
  • Data items can be inserted or deleted at any point in the list.
  • Can hold different data types.
  • Lists are dynamic (not fixed/static) and can grow and shrink as required.
